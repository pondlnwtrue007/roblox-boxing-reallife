# 🥊 Roblox Boxing — Real Life

เล่นเกม **Untitled Boxing Game** บน Roblox ด้วยการ **ชกมวยจริงหน้าเว็บแคม**
ใช้ **MediaPipe** จับท่าทางร่างกาย แล้วส่งคีย์บอร์ดเข้าเกมผ่าน **pydirectinput** (scancode)

![Python](https://img.shields.io/badge/Python-3.10%2B-blue)
![Platform](https://img.shields.io/badge/Platform-Windows-lightgrey)
![UI](https://img.shields.io/badge/UI-CustomTkinter-2b6cb0)
![License](https://img.shields.io/badge/License-MIT-green)

> แย็บ ฮุค การ์ด หลบ (ซ้าย/ขวา) ไม้ตาย และสกิล — ตรวจจับจากร่างกายจริง ไม่ต้องแตะคีย์บอร์ด

## ✨ Features

- 🎥 **หน้าตาแอปโมเดิร์น** (CustomTkinter ธีมมืด) — กล้อง + skeleton + มิเตอร์สด + ปุ่มควบคุมครบ
- 🥋 **8 ท่า** — แย็บ / ฮุค / การ์ด / หลบ (ตรง–ซ้าย–ขวา) / ไม้ตาย / สกิลท่า T
- 🎬 **จูนด้วย Record** — กดปุ่มแล้วทำท่าจริง ระบบตั้ง threshold ให้เอง ไม่ต้องเดาตัวเลข
- 📊 **มิเตอร์สด** — เห็นค่าจริงเทียบเส้น threshold ทุกท่า (เขียว = ท่าจะติด)
- 🎯 **เลือกหน้าต่างเป้าหมาย** — dropdown กันปุ่มรั่วไปแอปอื่น ส่งเฉพาะตอน Roblox โฟกัส
- 💾 **จำค่าจูน** — เซฟลงไฟล์อัตโนมัติ จูนครั้งเดียวจบ
- 🛡️ **ปลอดภัย** — เริ่มที่โหมด DRY-RUN (ไม่ส่งปุ่ม) กด LIVE เมื่อพร้อม

## 🎮 ท่า → ปุ่ม

| ท่าทางจริง | ตรวจจับจาก | ปุ่ม |
|-----------|-----------|------|
| แย็บซ้าย | แขนซ้ายเหยียดออก | `K` |
| ฮุคขวา | แขนขวาเหยียดออก | `L` |
| ตั้งการ์ด | ข้อมือสองข้างระดับหน้า ชิดกัน | `F` (ค้าง) |
| หลบ (พุ่งหน้า) | ย่อตัว หัวตรง | `Space` |
| หลบซ้าย / ขวา | ย่อ + เอียงหัวซ้าย/ขวา | `Space`+`A` / `Space`+`D` |
| ไม้ตาย | ยกแขนสองข้างขึ้นเหนือหัว | `Q` |
| สกิล (ท่า T) | กางแขนสองข้างระดับไหล่ | `R` |

## 📦 ติดตั้ง

ต้องมี **Python 3.10+** บน **Windows** และเว็บแคม

```bash
pip install -r requirements.txt
```

Dependencies: `mediapipe`, `pydirectinput`, `customtkinter`, `pillow`, `pygrabber`
(mediapipe ลาก opencv + numpy มาให้เอง · tkinter เป็น stdlib)

> โมเดลท่าทาง `pose_landmarker_lite.task` แนบมาในโปรเจกต์แล้ว (ถ้าลบจะโหลดใหม่อัตโนมัติ)

## 🚀 เริ่มใช้งาน

```bash
python app.py
```

จะเปิดแอป GUI ขึ้นมา:

1. **เลือกหน้าต่างเป้าหมาย** — ใน dropdown "หน้าต่างเป้าหมาย" เลือก **Roblox**
   (ระบบเลือก Roblox ให้อัตโนมัติถ้าเปิดอยู่ · กด 🔄 รีเฟรชถ้าเพิ่งเปิดเกม)
2. **Calibrate** — กดปุ่ม 🎯 แล้วยืนนิ่งในท่าปกติ ~2 วิ
3. **จูนท่า** — กดปุ่ม Record แต่ละท่า (ดูหัวข้อถัดไป) — ทำครั้งเดียวจำไว้
4. ทดสอบในโหมด **DRY-RUN** (สวิตช์ปิด) — ดูมิเตอร์/ป้ายท่าว่าตรวจถูกไหม โดยยังไม่ส่งปุ่ม

### ▶️ ส่งปุ่มเข้าเกมจริง

5. เปิด Roblox เข้าเกม → เปิดสวิตช์ **"ส่งปุ่มเข้าเกม"** (เป็น **LIVE** สีแดง)
6. คลิกไปที่หน้าต่าง **Roblox ให้โฟกัส** — แถบสถานะจะเป็น **🟢 ส่งเข้าเกมได้** → ชกได้เลย
   - ถ้ายังไม่โฟกัส Roblox จะเป็น **🟠 ยังไม่โฟกัส** (ปุ่มไม่ส่ง กันรั่วไป OBS/เบราว์เซอร์)
   - วางหน้าต่างแอปไว้จอที่ 2 หรือมุมจอ จะเห็น skeleton + มิเตอร์ระหว่างเล่น

> 💡 **เช็คว่าปุ่มออกจริงไหม:** เลือก "(ทุกหน้าต่าง)" ใน dropdown → เปิด Notepad โฟกัสไว้ →
> เปิด LIVE → ทำท่า จะเห็นตัวอักษร `k` `l` `f` พิมพ์ลง Notepad

> รุ่นเบา/ดีบักด้วยหน้าต่าง OpenCV ล้วน: `python main.py`

## 🎬 จูนด้วย Record (แนะนำ)

แทนการเดา threshold เป็นเลข ให้ **กดปุ่มแล้วทำท่านั้นจริงๆ** ~3 วิ ระบบจับ "ช่วงค่าจริง"
ของร่างกายคุณแล้วตั้งค่าให้เอง (เซฟลงไฟล์อัตโนมัติ) — กดจากปุ่มในแอป หรือคีย์ลัด:

| ปุ่ม | ท่าที่ต้องทำ (ค้างไว้/ทำซ้ำตามจอบอก) |
|:---:|---|
| `1` | **หมัด** — ชกซ้าย/ขวาสลับกัน สุดแรง |
| `2` | **หลบ** — ย่อตัวลง-ขึ้นซ้ำๆ (calibrate ก่อน) |
| `3` | **ไม้ตาย** — ยกแขนสองข้างสูงสุด |
| `4` | **การ์ด** — ตั้งการ์ดค้าง |
| `5` | **ท่า T** — กางแขนสองข้าง |

**มิเตอร์สด** ในแอปบอกค่าปัจจุบันเทียบ threshold — **เขียว = เกินเส้น (ท่าจะติด)**,
น้ำเงิน = ยังไม่ถึง เห็นทันทีว่าทำไมท่าไม่ติด

ค่าจูนเก็บที่ `%LOCALAPPDATA%\RobloxBoxingReallife\boxing_settings.json` (ลบ = คืนค่า default)

## 🥋 หลบมีทิศ (สลิปแบบมวย)

ตอน **ย่อหลบ** เอียงหัวไปทางที่จะหลบ:
หัวตรง → `Space` · เอียงซ้าย → `Space`+`A` · เอียงขวา → `Space`+`D`
(ดูค่า **LEAN** สดในแอป · ถ้าซ้าย/ขวากลับข้างตั้ง `DODGE_SWAP_LR = True` ใน `config.py`)

## 🛠️ จูนละเอียด (`config.py`)

ค่าทั้งหมดอยู่ด้านบนไฟล์ มีคอมเมนต์ครบ หน่วยเป็น "สัดส่วนของความยาวลำตัว"
(ใช้ได้เหมือนกันไม่ว่าตัวใหญ่/เล็ก กล้องใกล้/ไกล) จุดที่มักปรับ:

- **หมัดยิงรัวไป** → เพิ่ม `PUNCH_COOLDOWN_SEC`
- **แย็บ/ฮุคสลับข้าง** (ภาพกระจก) → `SWAP_LEFT_RIGHT = True`
- **record แล้วไว/ด้านไป** → ปรับ `REC_*_FRAC`

> **มุมกล้อง:** หมัดพุ่งตรงเข้ากล้องตรวจยากใน 2D — ยืน **เฉียงกล้องนิดหน่อย** หรือชกออกด้านข้าง
> เล็กน้อยจะจับได้ชัดกว่า (หมัดวัดจาก "การเหยียดแขน" ไม่ใช้เงื่อนไขความเร็วแล้ว = ตรวจง่าย)

## 🗂️ โครงสร้างไฟล์

| ไฟล์ | หน้าที่ |
|------|---------|
| **`app.py`** | แอป GUI (CustomTkinter) — ตัวหลัก |
| `main.py` | รุ่นเบา หน้าต่าง OpenCV + ปุ่มลัด |
| `config.py` | ค่าตั้งค่า/threshold ทั้งหมด |
| `pose_detector.py` | หุ้ม MediaPipe Pose → landmark + เมตริกท่าชกมวย |
| `motion_logic.py` | สมอง — ตัดสินท่า (priority + cooldown + hysteresis + record) |
| `input_sender.py` | ส่งปุ่มผ่าน pydirectinput (tap / hold / combo) |
| `settings_store.py` | เซฟ/โหลดค่าจูนลง JSON |
| `window_picker.py` | dropdown เลือกหน้าต่าง (สำหรับ `main.py`) |
| `camera.py` | กล้องแบบ threaded (ลด latency) |
| `winfocus.py` | เช็ก/ลิสต์หน้าต่าง active (กันปุ่มรั่ว) |
| `paths.py` | จัดการ path (รองรับ build `.exe`) |

---

## 🧠 ทำงานยังไง (ย่อ)

```
เว็บแคม → MediaPipe Pose → เมตริก (normalize ด้วยความยาวลำตัว)
        → ตัดสินท่า (priority: ไม้ตาย > ท่า T > การ์ด > หลบ > หมัด)
        → pydirectinput ส่ง scancode เข้า Roblox (เฉพาะตอนเกมโฟกัส)
```
